# Nourix

## Overview
Nourix is a wellness app with a React frontend and an Express backend.
AI responses are generated by Amazon Nova via Amazon Bedrock.

## Local Setup
1. Install dependencies: `npm install`
2. Create a `.env` file in the project root and fill it using the variables listed below
3. Configure Amazon Bedrock access:
   - For real Nova responses, provide AWS credentials with env vars or shared `~/.aws` config
   - For local demo mode, set `AI_PROVIDER=mock`
4. (Optional) Set `VITE_API_BASE_URL` if your API runs on a different host
5. Run the dev server: `npm run dev`

## Vercel Deployment
This repo is set up for a Vercel SPA frontend plus a Node.js serverless API:
- Static frontend build: `dist`
- Serverless API entrypoint: `api/index.ts`
- SPA routing: `vercel.json` rewrites non-API routes to `index.html`

Recommended Vercel settings:
- Build command: `npm run build`
- Output directory: `dist`

Environment variables to set in Vercel:
- `AI_PROVIDER`
- `AWS_REGION`
- `NOVA_MODEL_ID`
- Any AWS credential source you use for Bedrock
- `AI_ALLOW_MOCK_FALLBACK` set to `true` if you want deployed AI routes to fall back to built-in mock responses when Bedrock is unavailable
- `APP_URL`
- `CORS_ORIGIN` only if you need extra cross-origin callers beyond the deployed app itself

SQLite behavior on Vercel:
- By default, the app stores SQLite data in `/tmp/novafit.db` when `VERCEL` is present
- `/tmp` is writable but ephemeral on Vercel, so user data resets across deployments and may reset across cold starts
- For persistent production data, replace SQLite with an external database

Live dashboard behavior on Vercel:
- The app attempts an `EventSource` live stream first
- If the function stream reconnects or drops, the frontend falls back to polling automatically

## API
POST `/api/ai/plan`

Request body:
```json
{ "prompt": "..." }
```

Response body:
```json
{
  "diet": [{ "time": "...", "meal": "...", "description": "..." }],
  "exercise": [{ "name": "...", "duration": "...", "intensity": "..." }],
  "medicine": [{ "name": "...", "time": "...", "dosage": "..." }],
  "hydration": "...",
  "motivation": "..."
}
```

POST `/api/ai/chat`

Request body:
```json
{
  "message": "...",
  "history": [{ "role": "user", "content": "..." }]
}
```

Response body:
```json
{ "message": "..." }
```

## Environment Variables
- `AI_PROVIDER` - AI provider selector, default `nova`
- `AWS_REGION` - AWS region for Bedrock, e.g. `us-east-1`
- `NOVA_MODEL_ID` - Amazon Nova model ID, e.g. `amazon.nova-lite-v1:0`
- `NOVA_MAX_TOKENS`
- `NOVA_TEMPERATURE`
- `NOVA_TOP_P`
- `NOVA_TIMEOUT_MS`
- `AI_ALLOW_MOCK_FALLBACK` - Defaults to `true`; when enabled, the app falls back to mock AI responses if Bedrock is unavailable
- `AWS_ACCESS_KEY_ID` - Optional local dev credentials
- `AWS_SECRET_ACCESS_KEY` - Optional local dev credentials
- `AWS_SESSION_TOKEN` - Optional local dev session token
- `AWS_PROFILE` - Optional shared credentials profile name
- `AWS_DEFAULT_PROFILE` - Optional fallback shared profile name
- `AWS_SDK_LOAD_CONFIG` - Set to `1` when using `AWS_PROFILE`
- `AWS_SHARED_CREDENTIALS_FILE` - Optional custom shared credentials file path
- `AWS_CONFIG_FILE` - Optional custom shared config file path
- `PORT`
- `CORS_ORIGIN`
- `RATE_LIMIT_WINDOW_MS`
- `RATE_LIMIT_MAX`
- `SQLITE_DB_PATH` - Optional SQLite file path override. On Vercel the default becomes `/tmp/novafit.db`
- `VITE_API_BASE_URL` - Optional frontend API base URL
- `APP_URL` - Optional deployed app URL used for CORS and runtime metadata

Local Bedrock auth can come from any of these sources:
- `AWS_ACCESS_KEY_ID` + `AWS_SECRET_ACCESS_KEY` (+ `AWS_SESSION_TOKEN` if needed)
- `AWS_PROFILE` or `AWS_DEFAULT_PROFILE` with shared `~/.aws/credentials` / `~/.aws/config`
- Container or web identity credentials supplied by the runtime

## AWS EC2 Deployment (Amazon Linux 2)
1. Launch an EC2 instance with Amazon Linux 2, open ports 22/80/443, and attach an IAM role with `bedrock:InvokeModel`.
2. Install Nginx:
```bash
sudo amazon-linux-extras install -y nginx1
sudo systemctl enable nginx
sudo systemctl start nginx
```
3. Install Node.js via nvm:
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
source ~/.bashrc
nvm install 20
```
4. Get the app running:
```bash
git clone <your-repo>
cd nourix
npm install
npm run build
npm run start
```
5. Install PM2 and run the server:
```bash
npm install -g pm2
pm2 start npm --name nourix -- run start
pm2 save
pm2 startup systemd
```
6. Configure Nginx as a reverse proxy:
```nginx
server {
  listen 80;
  server_name your-domain.com;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```
7. Enable HTTPS with Certbot:
```bash
sudo amazon-linux-extras install -y epel
sudo yum update -y --skip-broken
sudo yum install -y certbot
sudo certbot certonly
```
Update your Nginx SSL config to reference `/etc/letsencrypt/live/<your-domain>/`,
test with `sudo nginx -t`, and restart Nginx.
